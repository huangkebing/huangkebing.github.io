六、BeanDefinition

从官方文档中得知，bean的所有信息都存储在BeanDefinition中，当然实际上肯定是他的实现类来实现存储。接口中定义的两组成员变量，先来看看：

变量 说明
String SCOPE_SINGLETON = ConfigurableBeanFactory.SCOPE_SINGLETON; 单例作用域的作用域标识符
String SCOPE_PROTOTYPE = ConfigurableBeanFactory.SCOPE_PROTOTYPE; 原型作用域的作用域标识符

这两个变量是bean的作用域标识符，简单来说单例作用域就是不管获取多少次bean都是同一个实例；而原型作用域，每次获取都会创建一个新的实例

变量 说明
int ROLE_APPLICATION = 0; 角色定义，应用程序的主要bean，通常是用户定义的bean
int ROLE_SUPPORT = 1; 角色定义，通常是某些较大配置的支持部分
int ROLE_INFRASTRUCTURE = 2; 角色定义，基础设施，即内部工作的bean

这三个变量将bean划分为了3个角色，我们定义的bean通常就是ROLE_APPLICATION。

然后来看方法，方法大部分都是get和set方法，而这些方法大多都和配置相呼应。BeanDefinition有点像是我们的实体类，用于存放数据。

方法 说明

@Nullable String getBeanClassName(); 返回此 Bean 定义的 Bean 类名。试着调用了一下返回的是全类名
void setScope(@Nullable String scope); 覆盖此 Bean 的目标作用域，并指定新的作用域名称
@Nullable String getScope(); 返回此 Bean 的目标作用域
void setLazyInit(boolean lazyInit); 设置是否应延迟初始化此 bean。如果false，则 Bean 将在启动时由执行单例预先初始化的 Bean 工厂实例化
boolean isLazyInit(); 返回此 Bean 是否应该延迟初始化，即在启动时不急于实例化。仅适用于单例 bean
void setDependsOn(@Nullable String... dependsOn); 设置此 Bean 所依赖的要初始化的 Bean 的名称。Bean 工厂将保证这些 Bean 首先初始化
@Nullable String[] getDependsOn(); 返回此 Bean 所依赖的 Bean 名称。
void setAutowireCandidate(boolean autowireCandidate); 设置此 Bean 是否是自动注入到其他 Bean。请注意，此标志旨在仅影响基于类型的自动注入。如果按名称自动注入，即使指定的 Bean 该标记为false，仍会注入 bean
boolean isAutowireCandidate(); 返回此 Bean 是否是自动注入到其他 Bean 的候选者
void setPrimary(boolean primary); 设置此 Bean 是否为自动注入主候选项。如果有多个bean被匹配到，取该值为true的bean

void setFactoryBeanName(@Nullable String factoryBeanName); 指定要使用的工厂Bean的名称（如果有）
@Nullable String getFactoryBeanName(); 返回工厂Bean的名称
void setFactoryMethodName(@Nullable String factoryMethodName); 指定工厂方法（如果有）。此方法将使用构造函数参数调用，如果未指定任何参数，则不调用任何参数。该方法将在指定的工厂 Bean（如果有）上调用，或者作为本地 Bean 类上的静态方法调用。
@Nullable String getFactoryMethodName(); 返回工厂方法
ConstructorArgumentValues getConstructorArgumentValues(); 返回此 Bean 的构造函数参数值
boolean hasConstructorArgumentValues() 返回此 Bean 是否定义了构造函数参数值，5.0.2版本新增方法
MutablePropertyValues getPropertyValues(); 返回要应用于 Bean 的新实例的属性值
boolean hasPropertyValues() 返回此 Bean 是否定义了属性值，5.0.2版本新增方法
void setInitMethodName(@Nullable String initMethodName); 设置 init 方法的名称
@Nullable String getInitMethodName(); 返回 init 方法的名称
void setDestroyMethodName(@Nullable String destroyMethodName); 设置 destroy 方法的名称
@Nullable String getDestroyMethodName(); 返回 destroy 方法的名称
